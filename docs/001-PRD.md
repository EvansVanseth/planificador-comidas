# Planificación semanal 
Este app está pensada para eliminar la tarea de "pensar que comer".
Es una herramienta que prepara un planning de comidas elaborado según ciertos criterios a partir de las recetas habituales que sueles hacer.

## Recetas
El primer paso para usar la aplicación, es crear una la lista de recetas habituales.
El listado de recetas podrá filtrarse por distintos campos.

### Estructura de datos de una receta
Este listado corresponde a todos los datos que contendrá una receta.
- Nombre                      Nombre que le das a a receta (no puede repetirse, es la referencia de la receta)
- Cantidad de comensales      Para cuantos comensales esta pensada la receta
- Ingredientes                Listado de ingredientes que necesitas para la receta
| Los ingredientes será una lista simple sin repeticiones.
| Al escribir un ingrediente se sugerira un completado para evitar en la medida de lo posible duplicidades
- Tiempo de elaboración       Tiempo aproximado que se tarda en elaborar la receta
- Elaboración (opcional)      Descripción de los pasos para elaborar la receta. Sirve como guía, la app no lo usa por ahora (puede que más adelante si ponemos IA entonces sí)
- Etiquetas                   Las etiquetas es el corazón de la aplicación, es lo que realmente identifica el ser de la receta y que será usado por el planificador.
  
  - Momento del día           Indica para qué momento del día está pensada la receta
  | Al menos hay que elegir una
  | Si se elige más de una significa que la receta puede ser para cualquiera de esos momentos
    - Desayuno                
    - Comida                  
    - Cena
  
  - Formato                   Indica si el plato se sirve caliente o frío
  | Al menos hay que elegir una
  | Si se eligen ambas, significa que el plato puede elaborarse de ambas formas
    - Caliente                
    - Frio                    
  
  - Tipo de plato             Indica a grandes rasgos qué contiene o qué es el plato
  | Al menos hay que elegir una
  | Si se elige más de una significa que la receta sirve o cumple para cualquiera de ellas
    - Carne                   
    - Pescado
    - Legumbres
    - Pasta
    - Ensalada
    - Arroz
    - Dulce
  
  - Estilos de vida/Dieta     Indicaciones adicionales (Estas etiquetas son totalmente opcionales)
    - Bajo en calorías
    - Vegetariano
    - Vegano
    - Alérgenos (Definir cual)
    - Personalizada           Se pueden crear etiquetas personalizadas para filtrar recetas

## Planificador
El planificador es la herramienta que facilitará el trabajo al usuario. 
El planificador preparará un listado de platos para el conjunto de días elegido por el usuario.
El usuario deberá crear un esqueleto de planificación y el planificador automático rellenará las recetas para los días.
Por descontado que puede hacerse manualmente si se desea. Por ejemplo, para definir un plato un día concreto antes de que el planificador rellene el esto.

### Creas una planificación
Cada planificación contendrá los siguientes datos
- Cantidad de semanas           Indica la cantidad de semanas que se van a planificar
- Días                          La cantidad de semanas creará un listado de días
| Cada día tendrá la siguiente estructura de datos
  - Comensales Desayuno         Indica cuantos comeran ese día para desayunar
  - Comensales Comida           Indica cuantos comeran ese día para mediodía
  - Comensales Cena             Indica cuantos comeran ese día para cenar
  - Exclusiones Desayuno        Indica un listado de etiquetas que NO queremos que se incluyan para ese día para desayunar
  - Exclusiones Comida          Indica un listado de etiquetas que NO queremos que se incluyan para ese día para mediodía
  - Exclusiones Cena            Indica un listado de etiquetas que NO queremos que se incluyan para ese día para cenar
  - Preferencia Desayuno        Indica un listado de etiquetas que queremos que se incluyan para ese día para desayunar si puede ser
  - Preferencia Comida          Indica un listado de etiquetas que queremos que se incluyan para ese día para mediodía si puede ser
  - Preferencia Cena            Indica un listado de etiquetas que queremos que se incluyan para ese día para cenar si puede ser 
- Día de inicio                 Indica el día concreto que empieza la planificación, deberá ser un lunes

Si quisieramos que no planifique algún desayuno o comida, simplemente bastará con poner los comensales a 0.
Deberemos ofrecer al usuario herramientas para seleccionar y editar grandes cantidades de días. 
Deberá existir un selector rápido para la semana completa.

### Planificador automático
- Siempre planifica todos los momentos del día a no ser que estén los comensales en 0
- No repite tipo de plato ni plato si puede ser
- Medidor de tipo de formato (tendencia a frio - caliente)
  Se dispondrá de un selector que permitirá indicar al planificador la tendencia del formato de plato que se desea.
  El selector, permite elegir un % que va desde 0 (solo fríos) a 100% (solo calientes).
  Si el selector está al 50% se tenderá a que el conjunto total de recetas tenga mitad y mitad.
  Siempre realizará distribución, es decir nunca agrupara una semana completa de calientes y luego una de frios, tenderá a intercalarlos.


## Listar ingredientes necesarios
- De la planificación se deriva una lista de ingredientes necesarios.
- La lista se ordenará por nombre de ingrediente
- Para cada fila se incluirá para cuantos comensales/comidas (esta información da una idea de la cantidad necesaria)
- Para cada fila se podrá indicar la cantidad que tenemos en despensa.
| Hay que incluir un botón que permita marcarla como "tengo de todo"

## Lista de la compra
- De la lista de ingredientes necesarios se derivan los que hay que comprar que no estén en la despensa.
- La lista será una replica de la lista de ingredientes necesarios excluiendo las cantidades o el ingrediente que esté marcado que se dispone en la despensa. 
- Para cada fila se podrá indicar si se ha compraado, para hacer la "lista de la compra".
