# TFM — Planificador de Comidas

## Objetivo

El objetivo principal del trabajo de fin de máster es elaborar una aplicación real adaptada a
tu nivel de conocimientos y experiencia.

Un proyecto original que ante todo demuestre tus conocimientos adquiridos a lo largo del
máster, sea diferenciador y te aporte valor personal. Algo que disfrutes construyendo y te
sirva para ponerte a prueba y enfrentarte a ti mismo.

Sobre el tipo de software, eres libre si lo que quieres desarrollar es una web, app, utilidad,
herramienta, juego o lo que se te ocurra. Todo está permitido. Simplemente piensa también
en tus intereses profesionales.

No existe una idea mejor que otra ya que, desde el principio, este máster ha sido agnóstico
en cuanto a tecnología y tipo de software a desarrollar.

## Requisitos

Deberás entregar los siguientes recursos relacionados con el proyecto a desarrollar para
que el TFM sea revisado y calificado.

### 1. Documentación completa y detallada (`README.md`)

- [x] Descripción general del proyecto.
- [x] Stack tecnológico utilizado.
- [x] Información sobre su instalación y ejecución.
- [x] Estructura del proyecto.
- [x] Funcionalidades principales.
- [x] Usuario y contraseña de prueba (en caso de que el proyecto tenga un login).

### 2. Código Fuente

- [x] Repositorio público en GitHub. URL codigo fuente en readme.md 
- [-] Repositorio privado en GitHub → Sólo se permitirá la opción de un
      repositorio privado si está perfectamente justificado (deberás conceder
      permisos al email `mouredev@gmail.com`).

### 3. Despliegue o publicación del proyecto en funcionamiento

Recomendado siempre que sea posible.

- [x] URL o acceso al proyecto real (añádela también en la documentación).

### 4. Slides de presentación

Con Google Slides, PowerPoint, Canva, o equivalente.

- [ ] URL de acceso público a la presentación o documento adjunto junto al
      código (añádela también en la documentación).

### 5. Vídeo explicativo

Con captura de pantalla (YouTube, Drive, o equivalente).

- [ ] URL de acceso público al vídeo (añádela también en la documentación).
- [ ] Es obligatorio capturar en vídeo la pantalla mientras realizas la explicación.
- [ ] Es opcional mostrar la grabación de tu rostro con una cámara.

### Contenido obligatorio en el repositorio

- [x] La documentación.
- [x] La información del despliegue (puede ser dentro de la documentación).
- [ ] Las slides (puede ser como un enlace a la documentación o subir
      directamente el documento con dicha presentación).

## Entrega

Para enviar el TFM, y todo el material descrito en la sección de requisitos, debes utilizar el
formulario de entrega que se encuentra en la descripción de la lección correspondiente al
Proyecto Final.

En el formulario deberás cumplimentar:

- **Nombre completo del alumno.**
- **Email utilizado en la inscripción del máster.**
- **URL del repositorio de GitHub.**
- **URL de despliegue o publicación** (si existe).
- **URL de las slides.**
- **URL del vídeo.**
- **Usuario y Contraseña de prueba** (si tu proyecto tiene login).
